# Black Box Testing Documentation

Folder ini berisi seluruh rancangan kasus uji dan skenario pengujian dengan metode **Black Box Testing** (Pengujian Kotak Hitam). Pengujian ini berfokus pada fungsionalitas sistem tanpa harus mengetahui struktur internal kode program.

## Dokumen di dalam folder ini:
*   [Black Box Test Cases](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Black_Box_Testing/Test_Cases.md) - Rincian kasus uji untuk autentikasi, pemesanan, dan fungsionalitas lainnya.

## Teknik yang Digunakan:
1.  **Equivalence Partitioning:** Membagi domain input ke dalam kelas-kelas data untuk meminimalkan jumlah test case.
2.  **Boundary Value Analysis (BVA):** Menguji batas nilai input (maksimal, minimal, dan nilai di luar batas).
3.  **State Transition Testing:** Menguji perubahan status dalam sistem (misal: status booking dari *Pending* -> *Approved* -> *Completed*).
