# Grey Box Testing Documentation

Folder ini berisi seluruh rancangan kasus uji, skenario, dan metode pengujian **Grey Box Testing** (Pengujian Kotak Abu-abu). Pengujian ini menggabungkan teknik Black Box dan White Box dengan memanfaatkan pengetahuan parsial tentang arsitektur internal sistem (skema database, REST API contract, file log, dan query).

## Dokumen di dalam folder ini:

1.  **[README.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/README.md)**: File panduan ini.
2.  **[Pattern Testing.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/Pattern%20Testing.md)**: Pengujian keseragaman skema JSON API dan pembacaan pola log keamanan.
3.  **[Matrix_Testing.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/Matrix_Testing.md)**: Matriks penelusuran kebutuhan (*Requirements Traceability Matrix*) dan pemetaan pengujian CRUD database.
4.  **[Orthogonal_Array_Testing.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/Orthogonal_Array_Testing.md)**: Pengujian kombinatorial optimal menggunakan larik ortogonal untuk menghemat jumlah test case.
5.  **[Regression_Testing.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/Regression_Testing.md)**: Rencana dan skenario pengujian regresi pasca perubahan skema basis data atau perbaikan bug backend.
6.  **[Test Plan.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/Test%20Plan.md)**: Rencana pengujian Grey Box terfokus pada API, antrean antarmuka (queues), dan database.
7.  **[teknik pengujian.md](file:///c:/SQA-PROJECT/3_Software_Testing_Documentation/Grey_Box_Testing/teknik%20pengujian.md)**: Penjelasan teori dan teknik-teknik pengujian Grey Box yang diterapkan pada proyek ini.
